# 🔥 Paper Reading Club

Клуб всех заинтересованных в чтении и анализе статей из области распределёных СУБД.

## ❓Зачем

Чтобы расширять кругозор, углублять уже имеющиеся знания по теме, вырабатывать насмотренность к уже известным алгоритмам, подходам и практикам, которые можно применять в разработке.

## 📝 Неполный список тематик

- consistency protocols;
- transaction protocols;
- storage algorithms and data structures;
- concurrent algorithms and data structures;
- query optimisations;
- distributed computing;

Предложения статей к обсуждлению можно слать лично или через PR-ы.

## 🕘 Время

Утра нечётных четвергов 6:00 AM UTC (обсуждаемо).

## 📴 Как проходит

Все участники должны внимательно прочитать статью и разобраться с предложенными протоколом/алгоритмом/структурой данных/подходом/итд. В процессе может быть что-то не до конца понятно (не все статьи раскрывают достаточно даже основные идеи) -- это нормально. Одна из задач обсуждений разобраться всем с возникшими вопросами и глубже разобраиться с предметом статьи. В идеале чтобы каждый участник нашёл хотя бы 1-2 вопроса для обсуждения (не бывает идеальных статей, к которым нет никаких вопросов). Так же интересно обсудить возможные применения прочитанному.

Презентация докладчика 7-15 минут и дальнейшее обсуждение в рамках 1-1.5 часа.

В конце предлагается новая статья для рассмотрения.
